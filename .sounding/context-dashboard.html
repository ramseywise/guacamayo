Cost & Efficiency

Am I spending well? What's driving cost?

How to read this: Cost is measured in "cost units" — each unit is roughly the price of reading 1M input tokens. Output tokens cost 5× more than input, so a session that writes a lot costs far more than one that reads a lot. Cache hits reduce input cost to near zero. The key lever is fewer output tokens per session and higher cache rates.

Context Health

Is context under control? Are sessions staying lean?

The 150k cliff: Sessions that grow past 150k context tokens cost 5× more per turn because the overflow can't be cached. In early July, 66-75% of sessions were hitting this cliff. The goal is <30%. Compaction is the mechanism — it compresses prior conversation to reclaim context space. Autocompact was active briefly (~Jul 17-18) but currently requires manual invocation from terminal.
! Autocompact is not running. It was active around Jul 17-18 (visible in the compaction rate spike) but now requires daily manual invocation from terminal. This is a known friction point — see Experiments tab.
