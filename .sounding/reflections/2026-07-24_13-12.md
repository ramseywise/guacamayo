Review scope detection: lightweight vs full, auto-detected from branch prefix, commit types, and plan doc presence. This is the answer to "when do we run the full review?" that we hadn't codified yet. Worktree cleanup as a review stage (Stage 0.5) — checking for leftover agent worktrees and stale branches before the code review even starts. `make review` as a print target, not an automated runner — because review is judgment, not lint.
